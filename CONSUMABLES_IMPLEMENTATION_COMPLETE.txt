## 🎓 How It Works

### User adds consumable with dropdown location:
```
1. User opens consumables.php
2. PHP loads warehouse_locations via get_all_warehouse_locations()
3. Dropdown rendered with all locations
4. User selects location from dropdown
5. Form submitted with warehouse_location_id
6. save_consumable_item() saves:
   • warehouse_location_id → consumables.warehouse_location_id
   • tenant_id → $_SESSION['tenant_id']
7. Database stores reference to warehouse location (not text)
8. When viewing list, location displays formatted (warehouse + position)
```

### System enforces tenant isolation:
```
1. User from Company A logs in
2. Session set: $_SESSION['tenant_id'] = 1
3. Consumables page loads
4. Query: SELECT * FROM consumables
5. apply_tenant_filter() adds: WHERE tenant_id = 1
6. Only Company A consumables displayed
7. Warehouse locations filtered by Company A
8. Company B cannot see Company A consumables
```

### Database compatibility handled:
```
1. Connection established (SQLite or MySQL)
2. $GLOBALS['db_type'] set appropriately
3. Each function checks: if ($db_type === 'sqlite')
4. Proper timestamp functions used
5. Correct data fetching methods
6. Proper string escaping
7. All queries work on both databases
```

## 📞 Support & Troubleshooting

### Dropdown Empty?
- Check warehouse_locations table has data
- Verify is_active = 1 for locations
- Check current user's tenant has locations
- Run: `php validate_consumables_dropdown.php`

### Location Not Saving?
- Check warehouse_location_id is INTEGER
- Verify location exists in warehouse_locations
- Check session tenant_id is set
- Check database has tenant_id column

### Can't See Consumables?
- Verify you're logged in to correct company
- Check tenant_id in session
- Run: `php tenant_isolation_audit.php`
- Run: `php cleanup_tenant_data.php`

### SQLite Errors?
- Verify database is SQLite (not MySQL)
- Check all functions detect db_type
- Verify migrations executed
- Run: `php validate_consumables_dropdown.php`
