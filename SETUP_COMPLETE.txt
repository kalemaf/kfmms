## How to Use The Feature

### Creating a New Part with Equipment Association

**Step 1:** Go to Inventory → Parts Master  
**Step 2:** Click "New Part"  
**Step 3:** Fill required fields:
- Part Code (e.g., `BEARING-6208`)
- Part Name (e.g., `Ball Bearing 6208ZZ`)
- Part Type, Category, Unit (existing fields)

**Step 4:** Scroll down to **"Equipment Compatibility & Spares"** section  
**Step 5:** Check equipment you want to associate with this spare:
```
☐ Pump-101 (Centrifugal Pump, Model XYZ)
☐ Motor-200 (Electric Motor, Model ABC)
☑ Compressor-15 (Air Compressor, Model DEF) ← Selected
```

**Step 6:** Click "Save Part"  

**Result:** The part is now registered as a spare for Compressor-15 (and any other checked equipment)

### Editing Equipment Associations

**Step 1:** Go to Inventory → Parts Master  
**Step 2:** Click "Edit" on an existing part  
**Step 3:** Scroll to "Equipment Compatibility & Spares" section  
**Step 4:** The section shows currently-linked equipment:
```
Currently Associated Equipment:
[Green card] Pump-101  [Green card] Motor-200
```

**Step 5:** Modify by checking/unchecking equipment:
- Check new equipment → adds association
- Uncheck equipment → removes association

**Step 6:** Click "Save Part"

**Result:** Equipment associations updated

## Troubleshooting

### Issue: "Equipment Compatibility & Spares" section not showing

**Solution 1:** Clear browser cache
- Press `Ctrl+F5` or `Cmd+Shift+R`
- Refresh the page

**Solution 2:** Check file modification
- Verify file: `inventory/parts_master.php` exists
- Check that file size is > 50KB (not truncated)

**Solution 3:** Verify database connection
- Access: `http://your-server/free-cmms%200.04/TEST_FEATURE_NOW.php`
- Check all items show green ✓

### Issue: "Undefined function" error

**Solution:** Functions are in `libraries/inventory_manager.php`
- Verify file exists at correct path
- File should be > 20KB in size
- Check `config.inc.php` includes this file

### Issue: Equipment list doesn't load

**Solution 1:** Verify equipment exists
```sql
SELECT COUNT(*) FROM equipment WHERE status IS NOT NULL;
```
If count is 0, you need to add equipment first via Equipment module.

**Solution 2:** Check database tables
```sql
PRAGMA table_info(equipment);
```
Should return columns: id, description, manufacturer, model, status, etc.

### Issue: Cannot save part with equipment

**Solution 1:** Check database permissions
- Verify database file is writable
- Check SQLite database file permissions

**Solution 2:** Check error log
- Equipment_spares table might not exist
- Run TEST_FEATURE_NOW.php to diagnose

## Technical Details (For Admins)

### Database Schema
```
equipment
├─ id (PK)
├─ description
├─ manufacturer
├─ model
├─ serial_number
└─ status

equipment_spares
├─ id (PK)
├─ equipment_id (FK → equipment.id)
├─ part_id (FK → parts_master.id)
├─ part_name
├─ part_number
├─ quantity
└─ notes

parts_master
├─ id (PK)
├─ part_code
├─ part_name
├─ part_type
└─ ... (existing fields)
```

### API Functions (For Developers)

**Get Equipment List:**
```php
$equipment = get_equipment_list($connection, $active_only = true);
// Returns: Array of equipment with id, description, manufacturer, model
```

**Get Part's Associated Equipment:**
```php
$spares = get_part_equipment_spares($part_id, $connection);
// Returns: Array of equipment associated with this part
```

**Attach Part to Equipment:**
```php
attach_part_to_equipment($part_id, $equipment_id, $quantity, $connection);
// Creates/updates equipment_spares entry
```

**Save All Associations:**
```php
save_part_equipment_associations($part_id, $equipment_ids_array, $connection);
// Updates all associations for a part at once
```
