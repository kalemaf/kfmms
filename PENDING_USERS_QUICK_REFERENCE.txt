   5. Employees can now login

🔐 WHAT PASSWORDS ARE SHOWN
══════════════════════════════════════════════════════════════════════════════

✓ SHOWN in Pending Users Dashboard:
   • Temporary password (plain text)
   • Set at user creation by system
   • Random 12-character string
   • Examples: bt7*+ObFj3tF, aB3$xYzP2@km, etc.

✗ NOT SHOWN in Dashboard:
   • Password hash (encrypted, not readable)
   • User's permanent password (after they set it)
   • Any passwords for non-pending users

Note: Dashboard only shows users with must_change_password = 1


⏱️ WHEN USER DISAPPEARS FROM DASHBOARD
══════════════════════════════════════════════════════════════════════════════

User is REMOVED from pending list when:
   ✓ User logs in and sets permanent password
   ✓ Admin clicks "✓ Activate" button
   ✓ Admin runs password change completion manually

User stays in pending list until one of above happens.


🆘 TROUBLESHOOTING
══════════════════════════════════════════════════════════════════════════════

Q: I don't see "📋 Pending Users" tab
A: You need Developer or Admin role
   Check: Login with developer account first

Q: User not showing in dashboard
A: User appears only if must_change_password = 1
   Check if user was actually created in database

Q: Password copy button not working
A: Try manual copy (highlight and Ctrl+C)
   Check browser console for errors

Q: Can't reset password
A: Permission denied - need admin/developer role
   Check session and permissions

Q: Need to view old/deleted user passwords
A: Dashboard only shows ACTIVE pending users
   Check database backup for historical data
