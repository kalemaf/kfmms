                    BLANK PAGE FIX - MARCH 17, 2026
================================================================================

ISSUE REPORTED:
  When accessing: http://127.0.0.1:8000/list.php?query_name=HJ_RECENT
  Browser shows: BLANK PAGE (no content visible)

ROOT CAUSE:
  Three issues combined:
  1. list.php tries to update a "top.title" frame that may not exist (frameset error)
  2. Direct access to list.php without proper session/frameset structure
  3. No graceful redirect for users accessing directly without being logged in

================================================================================
                            SOLUTIONS APPLIED
================================================================================

FIX #1: Safe Frame Reference in list.php
──────────────────────────────────────────

Modified: list.php (line 254)

Before:
  <body onLoad='top.title.location.href="title.php?title=<?=$nav_title ?>";'>

After:
  <body onLoad='if(top.title) { top.title.location.href="title.php?title=<?=$nav_title ?>"; }'>

Effect: Page no longer crashes if frame doesn't exist; safely checks before accessing


FIX #2: Auto-Redirect for Direct Access
─────────────────────────────────────────

Modified: list.php (after line 19, added redirect logic)

New code:
  // If accessing list.php directly with query_name parameter but not logged in
  if (!empty($_GET['query_name']) && empty($_SESSION['group'])) {
      // Save query preference and redirect to login
      $_SESSION['query_name'] = $_GET['query_name'];
      header("Location: ./index.php");
      exit;
  }

Effect: Users accessing list.php?query_name=HJ_RECENT are redirected to proper login → application entry
